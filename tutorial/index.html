Responsive Layout Tutorial

Overview

This tutorial will introduce you to responsive web design principles and guide you through creating your first mobile-first layout. You'll learn why websites need to adapt to different devices and how to make that happen with CSS.

Learning Objectives

By the end of this tutorial, students will be able to:

  • Explain what responsive design is and why it matters
  • Identify common device categories and screen sizes
  • Write basic media queries using proper syntax
  • Apply mobile-first design principles
  • Create flexible layouts using Flexbox
  • Test responsive behavior using browser developer tools

Project Setup

You'll need:

  • index.html - Main HTML file with sample content
  • style.css - CSS styles file
  • A web browser with developer tools

Part 1: Understanding Mobile-First Design

Mobile-first means we start by designing for the smallest screen size first (mobile), then progressively enhance for larger screens. This approach ensures your website works well on all devices.

Why mobile-first?

  • Most web traffic comes from mobile devices
  • Easier to scale up than scale down
  • Better performance on mobile devices

Step 1: Set Up Your HTML Structure

First, let's create the basic HTML structure. Notice how we use semantic HTML elements:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Responsive Layout</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <header class="header">Header</header>
    <main class="hero">Hero Section</main>
    <section class="cards-container">
        <div class="card">Card 1</div>
        <div class="card">Card 2</div>
        <div class="card">Card 3</div>
    </section>
</body>
</html>

Key point: The viewport meta tag is crucial for responsive design!

Step 2: Mobile Styles First (Base Styles)

Now let's add our mobile styles. These will be our default styles that work on small screens:

/* Mobile First Styles (Base Styles) */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

body {
    font-family: Arial, sans-serif;
    padding: 20px;
    background-color: #f5f5f5;
}

.header {
    background-color: #7B4397;
    height: 60px;
    margin-bottom: 20px;
    border-radius: 5px;
    display: flex;
    align-items: center;
    justify-content: center;
    color: white;
    font-weight: bold;
}

.hero {
    background-color: #2E86AB;
    height: 120px;
    margin-bottom: 20px;
    border-radius: 5px;
    display: flex;
    align-items: center;
    justify-content: center;
    color: white;
    font-weight: bold;
}

.cards-container {
    display: flex;
    flex-direction: column;
    gap: 20px;
}

.card {
    background-color: #F18F01;
    height: 80px;
    border-radius: 5px;
    display: flex;
    align-items: center;
    justify-content: center;
    color: white;
    font-weight: bold;
}

Notice: We use flex-direction: column for mobile to stack items vertically.

Step 3: Add Desktop Styles with Media Query

Now we'll add styles for larger screens using a media query. We only need to override what changes:

/* Desktop Styles - Only override what changes */
@media (min-width: 768px) {
    .cards-container {
        flex-direction: row;
        gap: 20px;
    }
    
    .card {
        flex: 1;
    }
}

Key concept: We only change what's different on desktop - the cards now go horizontal!

Step 4: Understanding the CSS

Let's break down the key CSS properties:

  • Flexbox: display: flex makes container flexible
  • flex-direction: column (mobile) vs row (desktop)
  • gap: Adds space between flex items
  • flex: 1: Makes cards equal width on desktop
  • Media query: @media (min-width: 768px) targets tablets and desktop

Step 5: Testing Your Layout

To test your responsive layout:

  1. Open your HTML file in a web browser
  2. Resize the browser window to see the layout change
  3. Use browser developer tools (F12) to simulate different devices
  4. Test on actual mobile devices when possible
