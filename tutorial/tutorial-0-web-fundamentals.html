Part 1: Understanding How Websites Work

The Web Development Trinity

Every website you visit is built using three core technologies:

HTML (Structure)

What it does: Creates the content and structure of your webpage

Think of it as: The skeleton and organs of a body

Example: Headings, paragraphs, images, links

CSS (Styling)

What it does: Controls how your webpage looks and feels

Think of it as: The skin, clothing, and makeup

Example: Colors, fonts, spacing, layouts

Browser (Interpreter)

What it does: Reads your HTML and CSS, then displays the result

Think of it as: A translator that turns code into visual websites

Example: Chrome, Firefox, Safari, Edge

Part 3: Your First Webpage

Step 1: Create Your HTML File

Create a new file called index.html and add this basic structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Website</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <header>
        <h1>Welcome to My Website</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section id="home">
            <h2>Home Section</h2>
            <p>This is the main content area of my website.</p>
        </section>
        
        <section id="about">
            <h2>About Section</h2>
            <p>Learn more about me and my projects.</p>
        </section>
    </main>
    
    <footer>
        <p>&copy; 2025 My Website. All rights reserved.</p>
    </footer>
</body>
</html>

Understanding the HTML Structure

  • <meta name="viewport">: This makes your site mobile-friendly (very important!)
  • <link rel="stylesheet">: Connects your CSS file to your HTML
  • <header>, <main>, <footer>: Semantic HTML elements that give structure meaning
  • <nav>: Dedicated area for navigation links

Step 2: Create Your CSS File

Create a new file called style.css with these basic styles:

/* Reset default margins and padding */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

/* Basic body styles */
body {
    font-family: Arial, sans-serif;
    line-height: 1.6;
    color: #333;
}

/* Header styles */
header {
    background-color: #4d1979;
    color: white;
    padding: 1rem;
}

header h1 {
    margin-bottom: 1rem;
}

/* Navigation styles */
nav ul {
    list-style: none;
    display: flex;
    gap: 1rem;
}

nav a {
    color: white;
    text-decoration: none;
    padding: 0.5rem;
    border-radius: 3px;
}

nav a:hover {
    background-color: rgba(255, 255, 255, 0.2);

/* Main content */
main {
    padding: 2rem;

    margin: 0 auto;

    margin-bottom: 2rem;

/* Footer */

    background-color: #333;
    color: white;
    text-align: center;
    padding: 1rem;
}

Part 4: Understanding Browser Developer Tools

Browser developer tools are essential for web development. They let you inspect, test, and debug your websites.

How to Open Developer Tools

Method 1: Right-click
  1. Right-click anywhere on your webpage
  2. Select "Inspect" or "Inspect Element"
Method 2: Keyboard shortcut
  • Windows: F12 or Ctrl+Shift+I
  • Mac: Cmd+Option+I

Key Developer Tools Features

Elements Tab

What it shows: Your HTML structure

What you can do: Edit HTML live, see which CSS rules apply

Try this: Click on any element to highlight it on the page

Console Tab

What it shows: Error messages and warnings

What you can do: See if anything is broken on your page

Try this: Look for red error messages

Device Toggle (ðŸ“± icon)

What it shows: How your site looks on different devices

What you can do: Test mobile and tablet views

Part 5: Introduction to Mobile-First Thinking

Why Mobile-First Matters

Mobile-first design means starting with the mobile experience and then enhancing for larger screens. Here's why this matters:

The Reality of Web Usage
  • Over 60% of web traffic comes from mobile devices
  • Mobile users often have slower internet connections
  • Touch interfaces require different design considerations
  • Screen space is limited and precious

Mobile-First Design Principles

Content Priority

Put the most important information first. Mobile users want to find what they need quickly.

Touch-Friendly Design

Buttons and links need to be large enough for fingers (minimum 44px).

Fast Loading

Mobile connections can be slow, so optimize images and minimize code.

Simple Navigation

Complex menus don't work well on small screens. Keep it simple.

Part 6: Testing Your Setup

Quick Test Checklist

Follow these steps to make sure everything is working:

  1. Save both files in the same folder
  2. Double-click your HTML file - it should open in your browser
  3. Check the styling - you should see a purple header and white background
  4. Open developer tools and click the device toggle
  5. Test mobile view - try "iPhone 12" or "Galaxy S20"
  6. Check for errors in the Console tab

âœ… Success Indicators

If everything is working correctly, you should see:

  • A webpage with a purple header and navigation
  • No error messages in the Console
  • The page responds when you switch to mobile view
  • Links change color when you hover over them

Problem: "CSS not loading" (page looks plain)

Problem: "File won't open"

Solution: Make sure you saved the file with .html extension, not .txt
