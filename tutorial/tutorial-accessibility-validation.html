Overview

Web accessibility ensures that websites are usable by people with disabilities, while validation ensures your code meets web standards. This tutorial teaches you both - creating inclusive websites that work for everyone.

Learning Objectives

By the end of this tutorial, you will be able to:

  • Validate HTML and CSS for standards compliance
  • Implement proper semantic HTML structure
  • Use ARIA labels and roles effectively
  • Ensure keyboard navigation works throughout your site
  • Test your site with accessibility tools
  • Create inclusive form experiences

Why This Matters

Over 1 billion people worldwide have disabilities. Making your site accessible isn't just about compliance - it's about creating better experiences for everyone. Plus, many accessibility practices also improve SEO and usability for all users.

Step 1: HTML Validation

Valid HTML is the foundation of accessible websites. Let's ensure your markup follows web standards:

Essential HTML Document Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Descriptive Page Title - Site Name</title>
    <meta name="description" content="Brief, descriptive summary of page content">
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <a href="#main-content" class="skip-link">Skip to main content</a>
    
    <header>
        <nav aria-label="Main navigation">
            <!-- Navigation content -->
        </nav>
    </header>
    
    <main id="main-content">
        <!-- Main page content -->
    </main>
    
    <footer>
        <!-- Footer content -->
    </footer>
</body>
</html>

Validation Checklist:

  • ✓ Use W3C HTML Validator to check your markup
  • ✓ Include proper DOCTYPE declaration
  • ✓ Set language attribute on html element
  • ✓ Use semantic HTML elements (header, nav, main, footer)
  • ✓ Ensure all tags are properly closed

Step 2: Accessibility Fundamentals

Semantic HTML Structure

Using the right HTML elements for the right purpose is crucial for screen readers:

<!-- Use heading hierarchy properly -->
<h1>Main Page Title</h1>
<h2>Section Title</h2>
<h3>Subsection Title</h3>

<!-- Use lists for grouped content -->
<nav aria-label="Main navigation">
    <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
    </ul>
</nav>

<!-- Use buttons for actions, links for navigation -->
<button type="button" onclick="toggleMenu()">Open Menu</button>
<a href="/next-page">Continue Reading</a>

Image Accessibility

<!-- Descriptive alt text for informative images -->
<img src="chart-sales-data.jpg" 
     alt="Sales increased 25% from January to March 2025">

<!-- Empty alt for decorative images -->
<img src="decorative-border.jpg" alt="">

<!-- Complex images need additional description -->
<figure>
    <img src="complex-chart.jpg" 
         alt="Quarterly sales by region"
         aria-describedby="chart-description">
    <figcaption id="chart-description">
        Detailed breakdown: North region 40%, South region 35%, 
        East region 15%, West region 10%
    </figcaption>
</figure>

Step 3: ARIA Labels & Roles

ARIA (Accessible Rich Internet Applications) attributes help screen readers understand dynamic content:

Common ARIA Patterns

<!-- Navigation landmarks -->
<nav aria-label="Main navigation">...</nav>
<nav aria-label="Breadcrumb">...</nav>

<!-- Form labels and descriptions -->
<label for="email">Email Address</label>
<input type="email" 
       id="email" 
       aria-describedby="email-help"
       required>
<div id="email-help">We'll never share your email with others</div>

<!-- Expanded/collapsed content -->
<button aria-expanded="false" 
        aria-controls="menu-items"
        onclick="toggleMenu()">
    Menu
</button>
<ul id="menu-items" aria-hidden="true">
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
</ul>

<!-- Current page indicator -->
<nav aria-label="Main navigation">
    <ul>
        <li><a href="/" aria-current="page">Home</a></li>
        <li><a href="/about">About</a></li>
    </ul>
</nav>

Step 4: Keyboard Navigation

Ensure your site works perfectly with keyboard-only navigation:

Essential CSS for Keyboard Users

/* Skip link for keyboard users */
.skip-link {
    position: absolute;
    top: -40px;
    left: 6px;
    background: #000;
    color: #fff;
    padding: 8px;
    text-decoration: none;
    border-radius: 4px;
    transition: top 0.3s;
    z-index: 1000;
}

.skip-link:focus {
    top: 6px;
}

/* Visible focus indicators */
:focus-visible {
    outline: 2px solid #2196f3;
    outline-offset: 2px;
    border-radius: 4px;
}

/* Remove outline for mouse users, keep for keyboard */
:focus:not(:focus-visible) {
    outline: none;
}

/* Make interactive elements keyboard accessible */
[tabindex="-1"]:focus {
    outline: none;
}

/* Ensure focus is visible on custom buttons */
.custom-button:focus-visible {
    box-shadow: 0 0 0 3px rgba(33, 150, 243, 0.3);
}

Keyboard-Friendly Navigation Menu

<!-- Accessible hamburger menu -->
<nav aria-label="Main navigation">
    <button class="menu-toggle" 
            aria-expanded="false" 
            aria-controls="main-menu"
            aria-label="Toggle navigation menu">
        <span class="hamburger-line"></span>
        <span class="hamburger-line"></span>
        <span class="hamburger-line"></span>
    </button>
    
    <ul id="main-menu" class="nav-menu">
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
    </ul>
</nav>

Step 5: Testing Tools & Checklist

Recommended Testing Tools

  • HTML Validation: W3C Markup Validator
  • CSS Validation: W3C CSS Validator
  • Accessibility: WAVE Web Accessibility Evaluator
  • Color Contrast: WebAIM Contrast Checker
  • Screen Reader: NVDA (free) or built-in screen readers

Quick Accessibility Checklist

Test Your Site:

  • ☐ Can you navigate the entire site using only the Tab key?
  • ☐ Are all interactive elements focusable?
  • ☐ Do all images have appropriate alt text?
  • ☐ Do form inputs have associated labels?
  • ☐ Is the heading structure logical (h1, h2, h3...)?
  • ☐ Does the site work with screen readers?
  • ☐ Do colors meet contrast requirements (4.5:1 for normal text)?
  • ☐ Are error messages clear and helpful?

Ready-to-Use Code Snippets

Accessible Form Template

<form>
    <fieldset>
        <legend>Contact Information</legend>
        
        <div class="form-group">
            <label for="name">Full Name *</label>
            <input type="text" 
                   id="name" 
                   name="name" 
                   required 
                   aria-describedby="name-error">
            <div id="name-error" class="error-message" aria-live="polite"></div>
        </div>
        
        <div class="form-group">
            <label for="email">Email Address *</label>
            <input type="email" 
                   id="email" 
                   name="email" 
                   required 
                   aria-describedby="email-help email-error">
            <div id="email-help" class="help-text">We'll never share your email</div>
            <div id="email-error" class="error-message" aria-live="polite"></div>
        </div>
        
        <button type="submit">Send Message</button>
    </fieldset>
</form>

Accessible Card Component

<article class="card">
    <img src="image.jpg" 
         alt="Brief description of image content">
    <div class="card-content">
        <h3><a href="/article-link">Article Title</a></h3>
        <p>Brief description of the article content...</p>
        <p><a href="/article-link" aria-label="Read more about Article Title">Read more</a></p>
    </div>
</article>
