Your instructor needs to access your live Week 1 site for grading while you work on Week 2. This method preserves your Week 1 work safely AND uses it as the foundation for your Week 2 multi-page site.

Why Backup AND Branch AND Clone?

  • Safety first: Backup folder protects your work from any Git mistakes
  • Keeps Week 1 live: Your instructor can still access your Week 1 site for grading
  • Practical starting point: Use your Week 1 content as templates for Week 2 pages
  • Professional workflow: Learn Git branching like real developers
  • No wasted work: Your Week 1 content, structure, and images become your Week 2 foundation

Complete 4-Step Process: Backup + Branch + Clone + Develop

This method combines three essential strategies: backup your work safely, branch to keep Week 1 live, and clone your content to build Week 2 efficiently:

Step 1: Create a Backup Folder

First, create a permanent backup of your Week 1 work

üîß What You'll Do:

  1. In your repository folder, create a new folder called week1-backup
  2. Copy ALL your Week 1 files into this backup folder:
    • Your HTML file (probably index.html)
    • Your CSS file (if you have one)
    • Your images folder with all 4 screenshots
    • Any other files you created for Week 1
  3. Commit this backup folder to your repository

Step 2: Create a New Branch for Week 2

This keeps Week 1 live while you work on Week 2

üîß Using GitHub Desktop:

  1. Open GitHub Desktop and make sure your repository is selected
  2. Click on Current Branch at the top (probably says "main")
  3. Click New Branch
  4. Name your branch: week2-development
  5. Click Create Branch

üîß Using GitHub.com:

  1. Go to your repository on GitHub.com
  2. Click on the branch dropdown (probably says "main")
  3. Type week2-development in the text box
  4. Click Create branch: week2-development

Step 3: Clone Your Week 1 Files for Week 2

Now use your Week 1 content as the foundation for Week 2

üîß What You'll Do:

  1. Make sure you're in your week2-development branch
  2. Copy your existing Week 1 HTML file to create new pages:
    • Rename your current index.html to crap-analysis.html
    • Copy crap-analysis.html to create new index.html
    • Copy crap-analysis.html to create about.html
    • Copy crap-analysis.html to create contact.html
  3. Create a new style.css file for your Week 2 styling
  4. Keep your images folder - you'll use the same screenshots

üí° Smart Strategy:

By copying your Week 1 file to create all four pages, you get:

  • Same basic HTML structure on all pages
  • Your CRAP analysis content already in place
  • Consistent styling foundation to build on
  • All your images linked and working

Step 4: When Ready, Merge Week 2 to Main

Only do this when Week 2 is complete and ready to go live

üîß Using GitHub Desktop:

  1. Make sure all your Week 2 changes are committed and pushed
  2. Switch to the main branch
  3. Click Branch menu > Merge into current branch
  4. Select week2-development branch
  5. Click Merge
  6. Push to GitHub to update your live site

üîß Using GitHub.com:

  1. Go to your repository on GitHub.com
  2. Click Pull requests
  3. Click New pull request
  4. Set base: main and compare: week2-development
  5. Click Create pull request
  6. Click Merge pull request

  • Week 2 development: You can work safely in your development branch
  • Easy switching: You can switch between branches to see both versions
  • Professional workflow: You're learning real developer practices

Start developing your Week 2 multi-page site in the week2-development branch!
