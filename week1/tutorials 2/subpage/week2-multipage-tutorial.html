Week 2: Multi-Page Website Development Tutorial

Overview

This tutorial will guide you through creating a professional multi-page website using your existing banner homepage as a foundation. You'll learn to organize content across multiple pages, create navigation systems, and maintain consistent styling.

Learning Objectives

By the end of this tutorial, students will be able to:

  • Create multiple HTML pages with consistent navigation
  • Organize content logically across different page types
  • Link between pages using relative file paths
  • Apply consistent CSS styling across multiple pages
  • Enhance typography and visual hierarchy
  • Structure professional content for web presentation

Project Structure

Your final project will include these files:

Your Repository ├── index.html (Homepage with banner) ├── about.html (About/Bio page) ├── crap-analysis.html (Week 1 CRAP analysis) ├── contact.html (Contact information) ├── style.css (Shared styles for all pages) └── images/ ├── banner-image.png ├── screenshot1.png ├── screenshot2.png └── (other images)

GitHub Desktop Setup

Before starting any coding session:

  1. Open GitHub Desktop
  2. Check current branch (usually main)
  3. Commit any existing changes before starting new work

Best Practice: Make commits after completing each step in this tutorial. This creates a backup history and allows you to revert if needed.

Step 1: Update Homepage Navigation

Replace Placeholder Links

First, update your existing navigation from placeholder links to actual page links:

<nav>
    <ul>
        <li><a href="index.html">Home</a></li>   
        <li><a href="about.html">About</a></li>  
        <li><a href="crap-analysis.html">CRAP Analysis</a></li>  
        <li><a href="contact.html">Contact</a></li>  
    </ul>
</nav>

Key Points:

  • Use index.html for the home link (even though you could use just #)
  • File names should be lowercase with hyphens for spaces
  • These are relative links - they point to files in the same directory

Checkpoint: Save Navigation Changes

After updating your navigation:

  1. Save your index.html file (Cmd+S)
  2. Test in browser - links won't work yet, but should show new text
  3. Commit changes in GitHub Desktop: "Update navigation with actual page links"
  4. Push to origin to backup your work

Step 2: Create the About Page

Basic About Page Structure

Create a new file called about.html in your main directory:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>About - Your Name</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <nav>
        <ul>
            <li><a href="index.html">Home</a></li>   
            <li><a href="about.html">About</a></li>  
            <li><a href="crap-analysis.html">CRAP Analysis</a></li>  
            <li><a href="contact.html">Contact</a></li>  
        </ul>
    </nav>

    <main>
        <h1>About Me</h1>
        
        <h2>Background</h2>
        <p>Write a professional paragraph about yourself here. Include your academic background, interests, and what you hope to achieve in web development.</p>
        
        <h2>Education</h2>
        <p>Include information about your current studies, previous education, or relevant coursework.</p>
        
        <h2>Interests & Goals</h2>
        <p>Share what interests you about web development, design, or your field of study. What are your professional goals?</p>
        
        <h2>Skills & Learning</h2>
        <ul>
            <li>HTML & CSS (learning)</li>
            <li>Web Design Principles</li>
            <li>Add other relevant skills</li>
        </ul>
    </main>

    <footer>
        <p>Contact me: <a href="mailto:your.email@example.com">your.email@example.com</a></p>
    </footer>
</body>
</html>

Important Notes:

  • Navigation is identical across all pages for consistency
  • Same CSS file is linked to maintain consistent styling
  • Professional content - write as if this is a portfolio piece
  • No banner on subpages - only the homepage gets the banner

Checkpoint: Test Your About Page

After creating your about page:

  1. Save the about.html file
  2. Test navigation - click between Home and About pages
  3. Check styling - should look consistent with your homepage
  4. Commit changes: "Add about page with professional content"

Step 3: Create the CRAP Analysis Page

Transform Your Week 1 Analysis

Create crap-analysis.html and structure your Week 1 analysis professionally:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>CRAP Analysis - Your Name</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <nav>
        <ul>
            <li><a href="index.html">Home</a></li>   
            <li><a href="about.html">About</a></li>  
            <li><a href="crap-analysis.html">CRAP Analysis</a></li>  
            <li><a href="contact.html">Contact</a></li>  
        </ul>
    </nav>

    <main>
        <h1>CRAP Design Analysis</h1>
        
        <p><strong>Website Analyzed:</strong> [Name of website you analyzed]</p>
        <p><strong>URL:</strong> <a href="[website URL]" target="_blank">[website URL]</a></p>
        
        <h2>Contrast</h2>
        <img src="images/contrast-screenshot.png" alt="Screenshot showing contrast examples" width="100%">
        <p>[Your analysis of contrast on the website - 75-100 words]</p>
        
        <h2>Repetition</h2>
        <img src="images/repetition-screenshot.png" alt="Screenshot showing repetition examples" width="100%">
        <p>[Your analysis of repetition on the website - 75-100 words]</p>
        
        <h2>Alignment</h2>
        <img src="images/alignment-screenshot.png" alt="Screenshot showing alignment examples" width="100%">
        <p>[Your analysis of alignment on the website - 75-100 words]</p>
        
        <h2>Proximity</h2>
        <img src="images/proximity-screenshot.png" alt="Screenshot showing proximity examples" width="100%">
        <p>[Your analysis of proximity on the website - 75-100 words]</p>
        
        <h2>Conclusion</h2>
        <p>[Add a brief conclusion about the overall design effectiveness - 50-75 words]</p>
    </main>

    <footer>
        <p>Contact me: <a href="mailto:your.email@example.com">your.email@example.com</a></p>
    </footer>
</body>
</html>

Don't Forget Alt Text!

Every image needs descriptive alt text:

  • alt="Screenshot showing contrast examples"
  • alt="Homepage banner with navigation menu"
  • alt="Color scheme comparison on website"

This improves accessibility and SEO.

Checkpoint: Upload Images and Test

After creating your CRAP analysis page:

  1. Upload your screenshots to the images folder
  2. Update image paths in your HTML
  3. Test all images display properly
  4. Commit changes: "Add CRAP analysis page with images"

Step 5: Enhance Your CSS

Add Image Styling

Add these styles to your style.css file to improve image presentation:

/* Image styling for analysis screenshots */
.analysis-image {
    width: 100%;
    max-width: 800px;
    margin: 1rem 0;
    border: 2px solid #ddd;
    border-radius: 8px;
    box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}

/* Image captions */
.image-caption {
    font-style: italic;
    color: #666;
    text-align: center;
    margin-top: 0.5rem;
    font-size: 0.9em;
}

Enhance Typography

Add more sophisticated typography controls:

/* Enhanced typography */
h1 {
    font-size: 2.5em;
    margin-bottom: 1rem;
    color: #4d1979;
}

h2 {
    font-size: 1.8em;
    margin-top: 2rem;
    margin-bottom: 1rem;
    color: #4d1979;
    border-bottom: 2px solid #e0e0e0;
    padding-bottom: 0.5rem;
}

/* Improve link styling */
a {
    color: #4d1979;
    text-decoration: none;
    transition: color 0.3s ease;
}

a:hover {
    color: #6b2c91;
    text-decoration: underline;
}

/* Focus states for accessibility */
a:focus {
    outline: 2px solid #4d1979;
    outline-offset: 2px;
}

Checkpoint: Enhanced Styling

After adding enhanced CSS:

  1. Update your CRAP analysis images to use the new analysis-image class
  2. Test all pages - styling should be consistent and improved
  3. Check focus states - tab through navigation to test accessibility
  4. Commit changes: "Enhance CSS with image styling and typography"

Step 6: Content Review and Polish

Content Checklist

Review each page for professional presentation:

  • Grammar and spelling - proofread all text
  • Professional tone - write as if this is a portfolio piece
  • Consistent formatting - headings, paragraphs, lists
  • Working links - all internal and external links function
  • Image alt text - descriptive text for all images

Technical Checklist

  • HTML validation - proper DOCTYPE, head section, semantic structure
  • CSS organization - logical order, consistent formatting
  • File organization - proper naming, images in images folder

Final GitHub Submission

Before submitting your assignment:

  1. Final commit with all changes: "Complete multi-page website with professional content"
  2. Push to origin to update your GitHub repository
  3. Test GitHub Pages - make sure your site is live and working
  4. Submit the GitHub Pages URL in your assignment submission

Step 7: Testing and Validation

Browser Testing

Test your website in multiple ways:

  • VS Code Live Preview - test during development
  • GitHub Pages - test your live site
  • Different browsers - Chrome, Firefox, Safari

Accessibility Testing

  • Tab navigation - can you navigate using only the Tab key?
  • Alt text - are all images described?
  • Color contrast - is text readable against backgrounds?
  • Focus indicators - can you see what's focused?

Common Issues to Check

  • Broken links - especially to images or between pages
  • Missing images - check file paths and names
  • Inconsistent navigation - should be identical on all pages
  • CSS not loading - check the stylesheet link

Submission Requirements

Your completed assignment should include:

  1. Four HTML pages - index.html, about.html, crap-analysis.html, contact.html
  2. Enhanced CSS - building on your existing style.css
  3. Professional content - well-written, proofread text
  4. Working navigation - consistent across all pages
  5. Images with alt text - including your CRAP analysis screenshots
  6. GitHub repository - properly organized and pushed
  7. Live GitHub Pages site - working and accessible

This tutorial builds on fundamental HTML and CSS skills while introducing multi-page website development and content organization.
